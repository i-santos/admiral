# SWARM CLI SPEC

## Local Multi-Agent Development System

Version: `v0.1`

Goal:

```text
Criar um CLI chamado "swarm" que permita rodar múltiplos agentes de desenvolvimento em paralelo dentro de qualquer projeto usando git worktrees, task graph e scheduler local.
```

Cada projeto terá **seu próprio swarm isolado**, mas todos usarão **o mesmo CLI global**.

---

# 1. DESIGN PRINCIPLES

O sistema deve seguir estes princípios:

### 1️⃣ Local-first

Nenhum serviço externo necessário.

Tudo roda com:

```
filesystem
git
node
shell
```

---

### 2️⃣ Reutilizável entre projetos

O swarm deve ser:

```
CLI global
+
runtime local
```

---

### 3️⃣ Isolamento por projeto

Cada projeto possui seu próprio:

```
task graph
workspaces
logs
events
```

---

### 4️⃣ Execução paralela

Deve permitir:

```
3–10 agentes simultâneos
```

dependendo do hardware.

---

# 8. CLI COMMANDS

O CLI deve expor os seguintes comandos.

---

## swarm init

Inicializa swarm no projeto atual.

Cria:

```
.swarm/config.json
kanban/graph.json
kanban/board.json
workspaces/
logs/
events/
runtime/
```

---

## swarm run

Inicia scheduler.

Executa:

```
scheduler loop
spawn agents
monitor heartbeats
retry tasks
```

---

## swarm status

Mostra estado atual.

Exemplo:

```
TASK              STATUS     AGENT
backend-auth      running    agent-1
frontend-login    todo
tests-auth        blocked
```

---

## swarm task create

Cria nova task.

Exemplo:

```
swarm task create backend-auth
```

Adiciona ao:

```
kanban/graph.json
```

---

## swarm task list

Lista tasks.

---

## swarm task retry

Retry manual de task.

---

## swarm merge

Merge de task em review.

---

## swarm cleanup

Remove workspaces finalizadas.

# 9. SCHEDULER

Arquivo:

```
lib/core/scheduler.js
```

Responsabilidades:

```
load config
load graph
detect ready tasks
spawn agents
monitor heartbeats
retry tasks
recovery
```

Loop:

```
while running:

  load graph

  find ready tasks

  spawn agents

  monitor running agents

  update board

  sleep interval
```

---

# 10. READY TASK RULE

Uma task está READY quando:

```
status = todo
AND
all depends_on tasks = done
```

---

# 11. WORKSPACE MANAGER

Arquivo:

```
workspace-manager.js
```

Responsável por:

```
create workspace
setup sparse checkout
cleanup workspace
```

Criar workspace:

```
git worktree add workspaces/<task-id> -b agent/<task-id>
```

---

# 12. SPARSE CHECKOUT

Cada agente deve ver apenas seu escopo.

Backend:

```
src/backend
tests
```

Frontend:

```
src/frontend
tests
```

---

# 13. AGENT RUNNER

Arquivo:

```
agent-runner.js
```

Fluxo:

```
spawn agent
cd workspace
run codex
commit progress
mark review
```

Exemplo execução:

```
codex exec backend-agent
```

---

# 14. HEARTBEAT

Cada agente publica heartbeat em:

```
runtime/heartbeats/<agent>.json
```

Formato:

```json
{
  "agent": "agent-backend-1",
  "task_id": "backend-auth",
  "updated_at": "timestamp"
}
```

---

# 15. DEAD AGENT DETECTION

Se heartbeat expirar:

```
mark task failed
schedule retry
```

---

# 16. EVENT BUS

Arquivo:

```
events/events.log
```

Formato:

```
timestamp | event | task | agent
```

Eventos:

```
TASK_CREATED
TASK_CLAIMED
TASK_STARTED
TASK_DONE
TASK_FAILED
AGENT_DEAD
MERGE_COMPLETED
```

---

# 17. MERGE FLOW

Merge apenas via:

```
swarm merge <task>
```

Fluxo:

```
checkout main
merge agent branch
mark task done
cleanup workspace
```

---

# 18. RETRY POLICY

Config:

```
max_retries_per_task
```

Fluxo:

```
failure
↓
retry_wait
↓
todo
```

# 19. RECOVERY

Na inicialização:

```
detect dead agents
rebuild board
retry tasks
cleanup locks
```
